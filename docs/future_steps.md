# Аналіз проєкту: Можливі покращення та нові функції (Future Steps)

Цей документ містить стратегічний аналіз проєкту **Drone Topometric Localization System**, що виходить за межі існуючого `MODERNIZATION_PLAN.md`. Він фокусується на принципово нових можливостях, глибокій архітектурній оптимізації та інтеграції з реальними екосистемами БПЛА.

---

## 1. Покращення AI та Vision Pipeline

### 1.1 Перехід на SAM 2 (Segment Anything 2)
Наразі система використовує `YOLOv11-Seg` та розглядає `Efficient-SAM` для маскування динамічних об'єктів. **SAM 2** від Meta пропонує нативну підтримку відео-сегментації (zero-shot video object segmentation).
* **Суть**: Динамічні об'єкти можна трекати та маскувати більш стабільно з врахуванням часового контексту (temporal consistency), зменшуючи навантаження на обчислювач кожного окремого кадру.
* **Перевага**: Краща стійкість до незвичних транспортних засобів, камуфльованої техніки тощо, що не входять у стандартні класи COCO.

### 1.2 Робота в інфрачервоному (Thermal/IR) діапазоні
Дрони часто літають вночі або в умовах густого туману.
* **Суть**: DINOv2 та ALIKED можна донавчити або дистилювати (Knowledge Distillation) для мультимодальної роботи з Thermal-камерами (наприклад, через датасет [Približheniye] або власні зібрані дані).
* **Імплементація**: Додавання мультимодальної гілки в `FeatureExtractor`, що автоматично перемикається залежно від вхідного стріму.

### 1.3 Від Homography до 3D-PnP (Perspective-n-Point) з DEM
Поточна локалізація спирається на 2D-гомографію (проекцію площини на площину).
* **Проблема**: Гомографія деградує при сильних змінах висоти рельєфу (будівлі, гори).
* **Рішення**: Інтеграція Digital Elevation Models (DEM) — наприклад, SRTM або ALOS. При створенні БД витягувати висоту (Z) для кожної ключової точки. У режимі реального часу локалізація відбувається через вирішення 3PnP (PoseLib), що дасть 6-DoF координати (X, Y, Z, Roll, Pitch, Yaw).

## 2. Інтеграція з екосистемами БПЛА (ROS2 / MAVLink)

Наразі застосунок є ізольованим GUI-рішенням для перегляду та симуляції на відео. 
Для того, щоб система стала частиною апаратного пайплайну дрона, потрібні такі функції:

### 2.1 MAVLink / ROS2 Bridge
* **Суть**: Створення фонового сервісу, який діє як ROS2 Node або MAVLink Companion Computer.
* **Функція**: Замість простого відображення на Leaflet-карті, система постійно транслює повідомлення (`VISION_POSITION_ESTIMATE` в MAVLink або `nav_msgs/Odometry` в ROS2) у політний контролер (PX4 / ArduPilot).
* **Перевага**: Це дозволить дрону автоматично перемикатися на візуальну навігацію в зонах GPS-спуфінгу без втручання оператора.

### 2.2 Tightly-Coupled Visual-Inertial Odometry (VIO)
Під час різкого маневру або прольоту над "безтекстурними" зонами (вода, поле) DINOv2/ALIKED можуть тимчасово втрачати позицію.
* **Суть**: Інтеграція IMU-сенсорів (акселерометр + гіроскоп) з дрона для заповнення "сліпих зон" між візуальними виявленнями. Оптимізація цього через фільтр Калмана (ESKF) або розширений GTSAM.

## 3. Картографія та 3D Реконструкція

### 3.1 Генерація 3D-мапи (3D Gaussian Splatting)
Система вже має базу даних зображень, ознак (ALIKED) та оптимізованих поз камер.
* **Нова функція**: Використання збережених HDF5/LanceDB місій для автоматичного тренування об'єктів топографії методом **3D Gaussian Splatting (3DGS)** (суттєво швидше за NeRF).
* **Застосування**: Оператор після розвід-польоту може не лише передивлятись 2D-панораму, а й детально облетіти місцевість у повноцінному 3D, оцінити висоту перешкод.

### 3.2 Офлайн-карти (Offline Map Caching)
Поточний `MapWidget` через Leaflet вимагає інтернету для завантаження супутникових знімків.
* **Рішення**: Інтеграція сервісу кешування тайлів (MBTiles або GeoPackage).
* **UI інструмент**: У вікні місії додати кнопку "Завантажити супутникову мапу для обраного полігону", щоб заздалегідь підготувати систему для роботи у польових умовах без зв'язку.

## 4. Оптимізація та Ройовий інтелект (Edge & Swarm)

### 4.1 Swarm Collaborative Mapping (Роєве картографування)
* **Суть**: Розширення архітектури LanceDB для підтримки "хмарного" або P2P (через Mesh-мережу) обміну дескрипторами.
* **Функція**: Декілька дронів розвідують територію, один ділиться global descriptors з іншими. Якщо другий дрон потрапляє в зону, де був перший, він автоматично локалізує себе за його картою без необхідності окремого етапу калібрування.

### 4.2 Спеціалізована оптимізація під NVIDIA Jetson
* **Суть**: Використання JetPack SDK API. 
* Замість OpenCV для кадрування відео — використання нативного `NvMedia` для zero-copy передачі кадрів (GPU-to-GPU) безпосередньо з ISP сенсора в TensorRT engine DINOv2 / YOLO. Це знизить споживання CPU майже до нуля і розвантажить шину пам'яті.

## Резюме пріоритетів

1. **High Priority (швидке впровадження + висока цінність)**:
   * Офлайн-карти та кешування тайлів.
   * MAVLink трансляція `VISION_POSITION_ESTIMATE`.
2. **Medium Priority (requires math update)**:
   * 3D-PnP з DEM (перехід від гомографії до 6-DoF + врахування рельєфу).
   * Інтеграція IMU.
3. **Long-term Vision (дослідження)**:
   * Thermal/IR донавчання базових моделей.
   * 3D Gaussian Splatting як результат збереженої БД.
