# Новітні аналоги методів у Drone Topometric Localization

Цей документ містить огляд найостанніших методів Machine Learning та Computer Vision (станом на 2024-2025 роки), які є прямими еволюційними замінниками для алгоритмів, що наразі використовуються у проєкті.

---

## 1. Локальні ознаки та зіставлення (Local Feature Matching)
### Тепер: `ALIKED + LightGlue`
ALIKED — швидкий детектор, а LightGlue є індустріальним стандартом (SOTA 2023) для розрідженого матчингу (sparse matching). Проте пайплайн "детекція → опис → матчинг" має обмеження у зонах з малим тестуванням (вода, сніг).

### Новітні аналоги:
*   **MASt3R (Matching and 3D Reconstruction) / DUST3R (2024):** 
    *   **Суть:** Кардинально новий підхід від Naver Labs. Замість того, щоб знаходити точки і спаровувати їх, модель одночасно проектує обидва зображення у спільний 3D-простір. Вона видає щільну (dense) 3D-карту збігів і відносні пози камер без RANSAC. 
    *   **Перевага для проєкту:** Відпадає проблема "немає достатньої кількості inliers". Працює навіть при екстремальних змінах ракурсу.
*   **DeDoDe + RoMa (2024):**
    *   **Суть:** `DeDoDe` (Detector and Descriptor) розділяє навчання визначення "місця" (keypoint) і "його суті" (descriptor), на відміну від спільних моделей як ALIKED чи SuperPoint. У комбінації з `RoMa` (Robust Matcher), цей пайплайн виграв змагання Image Matching Challenge 2024.
*   **OmniGlue (Google, 2024):**
    *   **Суть:** Генералізована foundation model для формату графового матчингу, що може працювати з "будь-якими" екстракторами як вхідними даними.

## 2. Глобальний пошук (Visual Place Recognition - VPR)
### Тепер: `DINOv2 (ViT-L/14) + FAISS зіставлення`
DINOv2 використовується zero-shot, де глобальний дескриптор — це просто CLS токен або усереднення патчів.

### Новітні аналоги:
*   **AnyLoc (2024):**
    *   **Суть:** Універсальний підхід, який пропускає ознаки фундаментальних моделей (DINOv2 / CLIP) через шар агрегації (VLAD — Vector of Locally Aggregated Descriptors). 
    *   **Перевага:** Керує семантикою: дозволяє ігнорувати небо, дерева або автомобілі під час побудови дескриптора.
*   **SelaVPR (2024) / SALAD:**
    *   **Суть:** Використовує легкі адаптери поверх замороженого DINOv2. Замість того, щоб вчити модель "з нуля", SelaVPR тюнить лише тонкий шар, зважаючи на геометрію. SALAD використовує алгоритм Сінкхорна (виділяє сміттєві ознаки в "dustbin").

## 3. Маскування динаміки (Сегментація)
### Тепер: `YOLOv11-Seg`
YOLO добре сегментує поточні об'єкти (автомобілі на дорозі), але обробляє кожен кадр незалежно.

### Новітні аналоги:
*   **SAM 2 (Segment Anything 2) (Meta, серпень 2024):**
    *   **Суть:** Це foundation model для **відео**. Замість сегментації статичного кадру, SAM 2 має пам'ять (memory attention) і будує маску в часі (spatial-temporal track). 
    *   **Перевага:** 100% захист від "мерехтіння" масок між кадрами та здатність маскувати невідомі транспортні засоби, яких немає у навчальній вибірці YOLO (zero-shot generalization).
*   **Cutie (2024) / XMem:**
    *   **Суть:** Швидкі моделі Video Object Segmentation (VOS), які потребують маски лише на першому кадрі, а далі з мінімальними витратами VRAM ведуть цей об'єкт.

## 4. Оптимізація графу та математика
### Тепер: `SciPy Levenberg-Marquardt (least_squares)` та `OpenCV MAGSAC++`

### Новітні аналоги:
*   **SymForce (Skydio):**
    *   **Суть:** Бібліотека від розробників автономних дронів Skydio. Вона бере вашу символьну математику мовою Python (включаючи матриці, повороти, SO3 групи) і генерує високошвидкісний оптимізований `C++` код.
    *   **Перевага:** Розв'язання нелінійної оптимізації може бути у **10-100 разів швидшим** за SciPy, не вимагаючи від вас писати на C++.
*   **VSAC (2024) або Differentiable RANSAC:**
    *   **Суть:** Набагато швидші за класичний MAGSAC++, орієнтовані на інференс у пайплайнах з нейромережами та динамічною оцінкою шуму.

## 5. Парадигма 2D vs 3D
### Тепер: `2D → 2D Гомографія` (Матриця 3x3)
Гомографія ідеальна для супутника, але передбачає, що поверхня плоска.

### Новітні аналоги (Перехід до Metric Depth):
*   **Depth Anything v2 (Metric Depth):**
    *   **Суть:** Раніше монокулярна глибина (з однієї камери) була відносною, зараз моделі фундаментальної глибини оцінюють **метричну відстань** (тобто прямо в метрах до пікселя).
    *   **Трансформація пайплайну:**
        1. Дрон робить знімок.
        2. `Depth Anything v2` миттєво генерує карту висот (глибини) в метрах.
        3. Коден піксель стає 3D-точкою (X, Y, Z).
        4. Замість обчислення гомографії, ви обчислюєте 3D PnP (Perspective-n-Point) алгебру.
    *   **Перевага:** Повна стійкість до рельєфу: гори, кар'єри, міська забудова більше не викликатимуть зсув геоприв'язки.
