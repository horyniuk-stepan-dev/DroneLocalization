# План реалізації: Відстеження об'єктів (Object Tracking) за допомогою YOLO

## 1. Мета та огляд (Goal & Overview)
**Мета:** Розширити фазу Tracking (локалізації дрона в реальному часі/з відео) можливістю у фоновому режимі відстежувати рухомі об'єкти (автомобілі, люди) на відео з дрона, розраховувати їхні глобальні координати за допомогою розрахованої матриці локалізації і відображати їх переміщення на супутниковій мапі в інтерфейсі (UI).

## 2. Зміни в архітектурі (Architecture Changes)

### Етап 1: Оновлення YOLO Wrapper (`src/models/wrappers/yolo_wrapper.py`)
Поточна реалізація YOLO використовується лише для генерації статичних масок (відсікання динаміки під час створення бази). YOLOv8/v11 має вбудовану систему трекінгу (ByteTrack/BoT-SORT), яка призначає постійні ID об'єктам.
* **Додати метод `track_objects(frame)`:** 
  ```python
  results = self.model.track(
      frame, 
      persist=True,  # зберігати стан між кадрами
      classes=[0, 2, 3, 5, 7],  # 0: person, 2: car, 3: motorcycle, 5: bus, 7: truck
      conf=0.45,
      tracker="botsort.yaml" # або bytetrack.yaml для швидкості
  )
  ```
* Метод має повертати словник знайдених об'єктів: `track_id, class_id, bbox [x1, y1, x2, y2], confidence`.

### Етап 2: Оновлення розрахункового циклу (`src/workers/tracking_worker.py`)
Під час фази Tracking система знаходить матрицю трансформації відео-кадру в координати опорного кадру (H), а потім — в глобальні координати (GPS/Метрики).
* Під час успішної локалізації кадру дрона, викликати `yolo_wrapper.track_objects(frame)`.
* Для кожного відстеженого об'єкта взяти **нижню центральную точку** його `bbox` (це точка контакту машини чи людини з землею: `x_center = (x1 + x2)/2`, `y_bottom = y2`).
* Використати знайдені матриці (Local -> Anchor -> Global Metric -> GPS) для проєктування пікселя кадру в GPS-координату на карті.
* Відправити сигнал з новими координатами об'єктів в UI: `self.objects_detected.emit(tracked_objects_list)`.

### Етап 3: Обробка та утримання стану в UI (`src/gui/mixins/tracking_mixin.py`)
* Зберігати словник активних об'єктів `self.tracked_entities = {track_id: {class, gps_lat, gps_lon, last_seen_time}}`.
* Оновлювати координати об'єкта, якщо він є у новому кадрі. Відфільтровувати старі об'єкти (видаляти через 3-5 секунд, якщо їх втрачено відео-трекером).

### Етап 4: Відмальовка на карті (`src/gui/widgets/map_widget.py` + `map_template.html`)
* Створити JavaScript шар на базі Leaflet або folium для рухомих об'єктів (окремо від маркерів дрона і якірних точок).
* Додати метод Python -> JS (`add_or_update_object_marker(id, lat, lon, type)`), який змінює іконки (наприклад, червона крапка для автомобіля, синя для людини) та плавно анімує їх переміщення.

## 3. Конфігурація (`config/config.yaml`)
Додати новий блок в налаштування, щоб це можна було увімкнути/вимкнути:
```yaml
tracking:
  object_tracking:
    enabled: true
    classes: [0, 2, 3] # 0: person, 2: car, 3: motorcycle
    confidence_threshold: 0.5
    tracker_type: "bytetrack.yaml" # bytetrack - швидший, botsort - точніший
```

## 4. Потенційні проблеми та шляхи їх вирішення (Risks)
1. **Продуктивність (FPS Drop):** Вмикання постійного YOLO tracking може забрати % ресурсів у FeatureMatcher.
   * *Вирішення:* Використовувати меншу модель (YOLOv11n/s) і обробляти трекінг через кожен 2-й або 3-й кадр, аппроксимуючи позицію на карті в проміжних кадрах.
2. **Точність проєкції (Projection Error):** Якщо дрон летить не чітко зверху вниз (Nadir), а під кутом, нижня точка bbox не буде ідеально рівною точці на землі (через перспективу 3D об'єктів на 2D площині бази).
   * *Вирішення:* Додати просту евристику зміщення в залежності від кута камери (якщо відома телеметрія) або вважати погрішність в 1-2 метри допустимою.
